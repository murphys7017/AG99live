# V2 当前实现状态与下一步

快照日期：2026-04-26

## 一句话状态

主链路已跑通；动态主轴重构已完成“后端 canonical profile + 同步协议 + 最小 Profile Editor UI”两轮基础建设，当前重点转为把编辑器能力继续扩展到 couplings / bindings，并推进 profile-driven compiler / prompt。

## 最新进展

### Adapter（AstrBot 插件）

- V2 协议链路稳定运行：`input/output/control/system/engine`。
- 动作链路支持主回复内联 `<@anim {...}>` 提取优先，realtime 二次生成作为兜底。
- realtime 当前主路径产物已收口到 `engine.motion_intent.v1`，后端负责提取与兜底，不再把它长期编译成前端执行 plan。
- 新增 `motion_prompt_instruction` 配置，可向 inline contract 和 realtime selector 注入动作风格/幅度偏好。
- 协议入口开始收紧：开发期要求消息类型、payload 字段、schema_version 严格对应，非法载荷会显式 warning / error。

### Frontend（Electron + Vue）

- 动作播放链路已增强：基于 `turn_id` 缓存计划并等待音频 `playing` 起播，无音频场景有超时兜底。
- 已支持 `soft handoff` 计划接续、同计划去重和高频重启节流，降低 nonce 风暴导致的抖动。
- 已落地 `ModelEngine`：前端可同时接收 `engine.motion_plan` 与 `engine.motion_intent`，并在本地完成 `intent -> plan -> playback`。
- 已落地 ModelEngine 表现倍率设置：全局强度默认 `1.35`，12 轴支持单独倍率，设置窗口可调并通过 Desktop snapshot 持久化。
- 动作实验室已升级为调参入口：记录最近 5 次真实播放 plan，可选中记录后手调 12 轴、回放手调结果、保存调参样本；记录来源限定为 Live2D 起播成功后的有效 plan。
- 参数执行容错增强：参数解析支持模型参数表回退匹配，轴缺失在阈值内可继续执行，`body_roll` 候选修正支持 `BodyAngleY`。
- 口型动态修正：移除 `mouth_open` 的强制 `Math.max` 钳制，避免动作口型被压扁。
- 动作实验室新增 `Profile Editor`：可查看并编辑当前模型的 `semantic_axis_profile`，并通过桌宠主窗口复用现有 WS 链路保存回后端。

### Dynamic Semantic Axis（本轮新增）

- 后端新增 `semantic_axis_profile` canonical store：
  - 文件位置：`live2ds/<model>/ag99/semantic_axis_profile.json`
  - 单一事实来源在后端模型目录，不再以辅窗本地缓存作为事实来源
- 已落地 profile 基础字段与约束：
  - `profile_id`
  - `revision`
  - `source_hash`
  - `last_scanned_hash`
  - `status`
  - `user_modified`
- 已落地保存冲突保护：
  - `expected_revision` 冲突硬失败
  - `source_hash` / stale 冲突硬失败
- 已落地协议闭环：
  - `system.model_sync` 下发 `semantic_axis_profile`
  - `system.semantic_axis_profile_save` 回写 profile
  - save 成功后由后端重发 `system.model_sync` 作为确认
- 已落地最小 UI：
  - Action Lab 内新增 Profile Editor 面板
  - 当前开放编辑字段：`label`、`control_role`、`description`、`usage_notes`、`soft_range`、`strong_range`
  - `parameter_bindings` 当前只读展示

### 架构收口方向

- 已确认新的职责边界：后端保留动作意图提取与二次请求兜底，不再长期承担执行 plan 编译；前端将新增 `ModelEngine` 承担 `intent -> plan -> playback`。
- 详细方案见《[动作链路职责重划与ModelEngine方案](./动作链路职责重划与ModelEngine方案.md)》。

## 协议使用现状

- `input.*`：`input.text`、`input.raw_audio_data`、`input.mic_audio_end` 等
- `output.*`：`output.text`、`output.audio`、`output.image`、`output.transcription`
- `control.*`：`control.turn_started`、`control.turn_finished`、`control.playback_finished`、`control.error` 等
- `system.*`：`system.server_info`、`system.model_sync`、心跳与历史
- `engine.*`：过渡期同时存在 `engine.motion_plan` 与 `engine.motion_intent`

## 已知问题 / 剩余工作

- 动作语义和参数语义的映射仍有提升空间，部分情绪下“眼部显著、非眼部较弱”。
- 音频、口型、动作三者虽已做时间轴协调，但在不同模型与不同语速下仍存在体感差异。
- supplementary 的可见性已改善，但仍需继续调权与风格收敛。
- 动作强度倍率已具备可调入口，但默认值和每轴建议值仍需要实机标定。
- 调参样本目前先保存在前端 snapshot，尚未接入后端 prompt/few-shot 自动替换流程。
- Profile Editor 的保存成功目前依赖后端回刷新的 `system.model_sync`，还没有单独 success ack。

## 下一步（建议顺序）

1. 继续做播放侧平滑微调（同轮多计划过渡窗插值）。

6. 增加回归用例：turn 级动作/音频对齐、去重节流行为、supplementary 回退策略。
