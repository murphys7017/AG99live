# V2 当前实现状态与下一步

快照日期：2026-04-26

## 一句话状态

主链路已跑通；动态主轴重构已完成“后端 canonical profile + 同步协议 + Profile Editor 主体编辑能力 + profile-driven prompt/compiler/runtime v2 最小闭环”。下一步重点是实机验证、参数标定、前端测试补齐，以及 Phase 8 清理固定 12 轴核心路径。

## 最新进展

### Adapter（AstrBot 插件）

- V2 协议链路稳定运行：`input/output/control/system/engine`。
- 动作链路支持主回复内联 `<@anim {...}>` 提取优先，realtime 二次生成作为兜底。
- realtime 当前主路径产物已切到 `engine.motion_intent.v2`，后端只负责基于 semantic profile 提取/兜底生成动作意图，不再编译前端执行 plan。
- 新增 `motion_prompt_instruction` 配置，可向 inline contract 和 realtime selector 注入动作风格/幅度偏好。
- 协议入口开始收紧：开发期要求消息类型、payload 字段、schema_version 严格对应，非法载荷会显式 warning / error。

### Frontend（Electron + Vue）

- 动作播放链路已增强：基于 `turn_id` 缓存计划并等待音频 `playing` 起播，无音频场景有超时兜底。
- 已支持 `soft handoff` 计划接续、同计划去重和高频重启节流，降低 nonce 风暴导致的抖动。
- 已落地 `ModelEngine`：前端可同时接收 `engine.motion_plan` 与 `engine.motion_intent`，并在本地完成 `intent -> plan -> playback`。
- 已落地 ModelEngine 表现倍率设置：当前保留会真正影响 `motion_intent.v2 -> parameter_plan.v2` 的全局强度倍率，仍通过 Desktop snapshot 持久化。
- `Profile Editor` 已从动作实验室拆出为独立辅助窗口；动作实验室保留动作预览与历史动作微调。
- 历史动作微调已恢复为 v2 主轴样本闭环：读取最近 5 次 `parameter_plan.v2` 播放记录，用户可手动微调 primary/hint 主轴、播放预览、保存样本，并选择是否同步给后端大模型作为 few-shot 参考。
- 参数执行容错增强：参数解析支持模型参数表回退匹配，轴缺失在阈值内可继续执行，`body_roll` 候选修正支持 `BodyAngleY`。
- 口型动态修正：移除 `mouth_open` 的强制 `Math.max` 钳制，避免动作口型被压扁。
- 独立 `Profile Editor` 可查看并编辑当前模型的 `semantic_axis_profile`，并通过桌宠主窗口复用现有 WS 链路保存回后端。
- `ModelEngine` 已支持 `engine.motion_intent.v2 -> engine.parameter_plan.v2` 编译。
- Live2D runtime 已支持按 `parameters[].parameter_id` 执行 `engine.parameter_plan.v2`。

### Dynamic Semantic Axis（本轮新增）

- 后端新增 `semantic_axis_profile` canonical store：
  - 文件位置：`live2ds/<model>/ag99/semantic_axis_profile.json`
  - 单一事实来源在后端模型目录，不再以辅窗本地缓存作为事实来源
  - 扫描出的未知参数会生成稳定的 `debug_*` semantic axis id，Live2D 原始参数 ID 保留在 binding 中
- 已落地 profile 基础字段与约束：
  - `profile_id`
  - `revision`
  - `source_hash`
  - `last_scanned_hash`
  - `status`
  - `user_modified`
- 已落地保存冲突保护：
  - `expected_revision` 冲突硬失败
  - `source_hash` / stale 冲突硬失败
  - 模型 hash 变化时先标记用户修改过的 profile 为 `stale`，不因当前参数表变化提前中断加载
- 已落地协议闭环：
  - `system.model_sync` 下发 `semantic_axis_profile`
  - `system.semantic_axis_profile_save` 回写 profile
  - `system.semantic_axis_profile_saved` 明确确认保存成功
  - `system.semantic_axis_profile_save_failed` 明确返回保存失败、错误码和 message
  - `system.model_sync` 仍用于刷新 canonical profile，但不再承担保存结果确认职责
- 已落地 Profile Editor 主体 UI：
  - Action Lab 内新增 Profile Editor 面板
  - 已支持编辑轴基础字段、正向/反向语义、`value_range`、`soft_range`、`strong_range`
  - 已支持编辑 `parameter_bindings`
  - 已支持新增、删除和编辑 `couplings`
  - 已支持按角色/文本筛选 axes，并批量把扫描出的 `debug` axes 提升为 `primary/hint/derived/runtime/ambient` 或降回 `debug`
  - 已支持新建自定义主轴草稿，保存前由用户显式补齐描述、使用说明、正/负语义、parameter binding，并确认当前轴配置
- 已落地 profile 严格校验：
  - 前端保存前汇总展示缺失字段、非法 range、重复 id、跨轴重复 binding、非法 coupling 数值等错误
  - 后端保存层拒绝非法 axis id、非有限数字、反向 range、越界 neutral/range、跨轴重复 binding、负 `scale/deadzone/max_delta`、非法 coupling 和 coupling cycle
- 已落地 profile-driven 执行链：
  - realtime prompt 只暴露 `primary/hint` axes，禁止 `derived/runtime/ambient/debug`
  - inline motion contract 已切到 `engine.motion_intent.v2`
  - v2 compiler 使用 profile 校验 axis、支持多跳 coupling 传播，并输出 concrete `parameter_plan.v2`
  - v2 runtime 只按 `parameter_id` 写入 Live2D 参数
  - v2 路径对非法 mode、越界 duration、重复 parameter、非法 source、不可写 parameter 均 hard fail
  - v2 路径对 unknown/forbidden/non-number 主参数按 30% 错误率阈值处理，未超过阈值时继续并记录 diagnostics，超过阈值 hard fail
  - v2 路径对语义值或 target 越界会夹到对应极限值并记录 warning

### 架构收口方向

- 已确认新的职责边界：后端保留动作意图提取与二次请求兜底，不再长期承担执行 plan 编译；前端将新增 `ModelEngine` 承担 `intent -> plan -> playback`。
- 详细方案见《[动作链路职责重划与ModelEngine方案](./动作链路职责重划与ModelEngine方案.md)》。

## 本轮已完成

- 动态主轴第一轮基础设施已完成：
  - 后端 canonical profile
  - `model_sync` 下发
  - save 协议与 revision/source hash 冲突保护
  - 固定模板外的扫描参数会以 `debug_*` 语义轴进入 profile，后续可人工提升控制角色
  - semantic axis id 与 Live2D parameter id 已解耦，为 `motion_intent.v2` 和 `parameter_plan.v2` 做准备
- 动态主轴第二轮 Profile Editor 主体能力已完成：
  - 独立 Profile Editor 辅助窗口
  - 辅窗经 `DesktopBridge -> PetDesktopView -> useAdapterConnection` 复用主链路保存
  - 支持编辑 `parameter_bindings`、`couplings`、正向/反向语义和基础范围字段
  - 支持按角色/文本筛选 axes、参数排除关键词、批量设置控制角色，以及新建主轴草稿
  - 保存前会在前端汇总错误；后端仍作为最终 schema 校验边界，非法 profile 不会被静默修正或保存
- 动态主轴第三轮执行链迁移已完成：
  - dedicated profile save ack/fail 协议
  - `motion_intent.v2` normalize 与后端生成
  - profile-driven realtime prompt 与 inline contract
  - profile-driven frontend compiler
  - `parameter_plan.v2` preview/player/runtime 写入链路
  - v2 runtime 严格拒绝缺失/重复/不可写/非法参数；target 越界时夹到 Live2D 参数 min/max 并记录 warning
- 现有固定 12 轴兼容工具已部分降级：
  - 设置页已移除旧 12 轴逐轴倍率，避免出现能调但不生效的兼容项
  - `MotionTuningPanel` 已恢复为 v2 历史动作微调与样本保存面板
  - `BaseActionPreviewPanel` 仍保留 v1 预览能力，等待下一轮继续清理

## 本轮未完成

- 还没有做前端自动化测试框架，当前 UI 仍主要依赖类型检查、构建和手工联调。
- 还没有完成真实 Live2D 模型上的 v2 动作效果实机验证。
- 还没有删除固定 12 轴核心路径，`BaseActionPreviewPanel`、preview/player/runtime 的 v1 兼容分支仍存在。

## 已知问题 / 剩余工作

- v2 runtime 已接通，但具体动作效果需要在真实模型上验证 `parameter_bindings[].output_range` 是否合理。
- 动作语义和参数语义的映射仍有提升空间，部分情绪下“眼部显著、非眼部较弱”。
- 音频、口型、动作三者虽已做时间轴协调，但在不同模型与不同语速下仍存在体感差异。
- supplementary 的可见性已改善，但仍需继续调权与风格收敛。
- 动作强度倍率当前只保留全局强度；后续若需要恢复更细粒度控制，应改为真正参与 v2 编译的动态主轴级设置，而不是继续沿用旧 12 轴兼容项。
- 已启用的调参样本会通过 `system.motion_tuning_examples_sync` 同步到后端，并优先进入 realtime selector few-shot prompt。
