## 背景

当前动作链路已经收口为：

```text
后端生成/提取 engine.motion_intent.v1
-> 前端 ModelEngine 编译 engine.parameter_plan.v1
-> Live2D Direct Parameter 播放
```

实测问题是：Live2D 模型对参数幅度不够敏感，12 轴值即使变化，视觉表现也可能偏弱。因此需要把“语义动作意图”和“模型表现强度”拆开调节。

## 目标

- 后端允许用户补充动作生成指令，影响大模型输出的动作意图。
- 前端允许用户确定性调节动作表现幅度，不依赖大模型每次都输出足够夸张的数值。
- 支持全局动作强度与 12 轴单独倍率。
- 所有调节都必须显式、可见、可持久化，不允许静默掩盖错误。

## 非目标

- 不把最终 plan 编译职责放回后端。
- 不在后端按模型参数做最终幅度映射。
- 不用随机兜底动作掩盖 12 轴播放失败。
- 不在前端补缺失 schema、缺失 emotion_label、非法轴值。

## 设置项设计

### 1. 后端 Prompt 指令

配置名建议：

```text
motion_prompt_instruction
```

位置：AstrBot 插件配置 `_conf_schema.json`。

用途：用户补充给大模型的动作生成偏好。

默认值建议：

```text
Live2D 表现需要比真人更夸张。非中性情绪下，请让头部、眼睛、嘴部或眉毛至少 2 到 3 个轴明显偏离 50，避免输出几乎无动作的 45 到 55 区间。
```

注入位置：

- 主回复内联 `<@anim ...>` contract prompt。
- realtime 二次请求 selector prompt。

处理规则：

- 只作为模型指令，不作为可靠保证。
- 需要限制长度，例如最多 800 字符。
- 为空时使用内置默认提示。
- 不允许通过该字段改变协议结构，只能影响动作风格和幅度。

### 2. 前端全局动作强度

配置名建议：

```ts
motionIntensityScale: number
```

默认值：

```ts
1.35
```

范围：

```ts
min: 0.5
max: 2.5
step: 0.05
```

含义：

```text
1.0 = 使用模型原始输出幅度
1.35 = 默认增强，适合 Live2D
2.0 = 明显夸张
0.5 = 减弱一半
```

### 3. 前端 12 轴单独倍率

配置名建议：

```ts
axisIntensityScale: Record<DirectParameterAxisName, number>
```

默认值：所有轴 `1.0`。

范围：

```ts
min: 0
max: 2.5
step: 0.05
```

含义：

```text
0 = 禁用该轴表现
0.5 = 该轴减弱一半
1.0 = 该轴原始强度
1.5 = 该轴增强
2.0+ = 该轴明显夸张
```

12 轴中文名建议：

| Axis | 中文名 | 说明 |
| --- | --- | --- |
| `head_yaw` | 头部左右转向 | 左右看、侧头方向 |
| `head_roll` | 头部左右歪斜 | 歪头、俏皮、疑惑 |
| `head_pitch` | 头部上下俯仰 | 点头、低头、抬头 |
| `body_yaw` | 身体左右转向 | 身体朝向变化 |
| `body_roll` | 身体侧倾摆动 | 身体左右倾斜 |
| `gaze_x` | 视线左右 | 眼球左右看 |
| `gaze_y` | 视线上下 | 眼球上下看 |
| `eye_open_left` | 左眼开合 | 左眼睁闭程度 |
| `eye_open_right` | 右眼开合 | 右眼睁闭程度 |
| `mouth_open` | 嘴巴张合 | 说话、惊讶、哈欠 |
| `mouth_smile` | 微笑幅度 | 笑容、嘴角情绪 |
| `brow_bias` | 眉毛情绪 | 皱眉、挑眉、情绪压力 |

## 编译公式

只对 expressive intent 应用强度：

```ts
finalValue = 50 + (rawValue - 50) * motionIntensityScale * axisIntensityScale[axisName]
finalValue = clamp(round(finalValue), 0, 100)
```

idle intent 默认不放大：

```ts
if (intent.mode === "idle") {
  keepRawAxes();
}
```

原因：idle 是平静/待机/弱动作，不应该被强度设置放大成明显动作。

## 执行顺序

建议 ModelEngine 编译顺序：

```text
normalize motion_intent
-> validate 12 axes / emotion / schema
-> expressive: apply intensity settings
-> idle deadzone 判定
-> timing 生成
-> supplementary 推导
-> 输出 engine.parameter_plan.v1
-> usePreviewMotionPlayer 播放
```

注意：强度放大应发生在 idle deadzone 判定之前。否则模型输出 55/56 这种弱 expressive intent 会先被判成 idle，放大不会生效。

## 兼容策略

- 旧配置不存在时使用默认值。
- localStorage 没有 ModelEngine 设置时使用默认值。
- legacy `engine.parameter_plan.v1` 预览不重新编译，不应用 ModelEngine intent 强度；它已经是最终 plan。
- 后端仍然只输出 intent，不因 prompt 设置恢复 plan 编译。

## 开发期错误策略

必须硬失败：

- 缺 `schema_version`
- 缺 `emotion_label`
- 缺完整 12 轴进入前端
- 轴值非 number
- 轴值越界
- selectedModel 缺失
- Live2D Direct Parameter 接口不存在

允许 warning 后继续：

- 后端 intent 归一化阶段缺轴补 50
- supplementary 绑定失败数量不超过阈值
- expressive intent 因所有轴仍在 deadzone 内被判为 idle，但必须 warning

## 当前审阅结论与风险

- 过高倍率可能导致模型抽搐或穿帮，需要默认值保守。
- 不同模型参数灵敏度差异大，未来可能需要按模型保存一份设置。
- 如果单轴倍率在 supplementary 推导前应用，会影响 supplementary 选择数量和强度；这是预期行为，但需要实测。
- slider 当前是即时广播，结构上可接受；如果设置窗口拖动时出现卡顿，再加节流。
- 设置归一化会把非法倍率压回合法范围，这是用户设置层的安全归一化，不用于掩盖协议错误；动作 payload 的缺字段、非法 schema、非法轴值仍然拒绝。

## 推荐默认方案

第一版推荐：

```text
motionIntensityScale = 1.35
axisIntensityScale[all] = 1.0
```

先不做按模型多配置，先做全局持久化。等确认多个模型都有需求，再扩展为 per-model profile。
