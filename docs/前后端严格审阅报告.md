# 前后端严格审阅报告（AG99live）

快照日期：2026-04-23  
审阅范围：`adapter/` + `frontend/` + 当前文档  
审阅方式：代码静态审阅 + 回归验证

## 结论摘要

链路已从“功能可用”进入“体验与鲁棒性收敛”阶段。  
本轮主要风险从“前后端不通”转为“语义质量与体感一致性”。

## 运行验证

- 后端：`python -m pytest adapter/tests -q` -> `50 passed`
- 前端：`npm run typecheck` -> 通过
- 前端：`npm run build:web` -> 通过

## 审阅结论（按优先级）

### P0（已缓解）动作计划重启风暴

现象：

- 同一轮次短时间连续到达计划时，动作频繁 stop/start，体感抖动。

当前状态：

- 已新增计划去重与重启节流。
- 已支持 soft handoff，减少硬切换。

残余风险：

- 不同但高频计划仍可能导致跳变，后续应加过渡窗插值。

### P1（已缓解）动作/音频时间轴失配

现象：

- `engine.motion_plan` 与 `output.audio` 原本独立触发，容易错位。

当前状态：

- 前端已做 `turn_id` 级时间轴协调：等音频 `playing` 再起播动作；无音频时超时兜底。

残余风险：

- 不同模型与语速下仍有细微体感偏差。

### P1（已缓解）参数匹配过严导致整计划失败

现象：

- 某一轴绑定失败会终止整计划，易触发 `missing_axis_parameter:*`。

当前状态：

- 参数绑定已增加模型参数表回退匹配。
- 轴缺失容忍已放宽（超过阈值才整计划失败）。

残余风险：

- 模型差异较大时仍需扩充映射与校准策略。

### P1（部分缓解）supplementary 参数不足

现象：

- `supplementary_params` 偏少时，非眼部变化弱。

当前状态：

- 已支持 `parameter_action_library` 失败时回退 `base_action_library`。
- 回退路径已支持宽松匹配，降低空结果概率。

残余风险：

- 情绪强度与风格稳定性仍需继续调优。

### P2（待推进）语义层与播放层分层治理

现象：

- “生成什么动作”（语义）与“如何平滑播放”（执行）尚未完全分层闭环。

建议：

- 语义层：模板约束、跨轮平滑、幅度边界。
- 播放层：过渡窗、重定时、去重节流策略参数化。

## 下一步建议

1. 增加播放侧过渡窗插值，继续降低计划切换跳变。
2. 增加语义层跨轮 EMA 或 bounded-delta，稳定情绪连续性。
3. 补时序回归：plan 先到、audio 先到、无 audio 文本回复。
4. 收紧 debug 接口策略（鉴权或默认关闭远程调用）。
