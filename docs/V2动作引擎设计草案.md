# V2 动作引擎设计草案

快照日期：2026-04-21

## 当前边界

- 动作引擎属于前端 runtime，不属于 adapter 主职责。
- adapter 提供扫描结果和低频语义输入，不做逐帧动作决策。
- `engine.*` 在当前阶段只作为协议入口和调试桥。

## 已落地基础

### 后端输入底座

- `model_sync` 已携带 `base_action_library`
- 动作库已具备：
  - 通道/家族分组
  - 原子动作候选
  - 规则初筛结果
  - 可选 LLM 严筛结果与分析元数据

### 前端实验能力

- Action Lab 可选择原子动作并生成 `motion_plan` 预览 JSON
- 可发送 `engine.motion_plan` 到 adapter 进行协议联调
- 当前不执行真实动作，仅完成“计划可构造 + 消息可通”闭环

## 目标执行链

1. 输入层：接收动作计划、模型能力、播放上下文
2. 编排层：生成时间轴（并行/串行、时长、强度）
3. 混合层：按通道合成参数目标值
4. 执行层：逐帧写入 Live2D 参数并收敛回 idle

## 最小可用执行器（MVP）

### MVP 支持范围

- 通道优先：`head/body/gaze/eye/brow/mouth/breath`
- 计划输入：Action Lab 生成的 `engine.motion_plan`
- 时间模式：`parallel` 和 `sequential`
- 控制参数：`duration_scale`、`intensity_scale`、`sequential_gap_ms`

### MVP 不做

- 文本语义自动编排
- 复杂韵律分析
- 自动 expression/motion fallback 决策
- 多计划并发冲突解算

## 协议约定（当前）

### 调试计划消息

- 入站类型：`engine.motion_plan`
- 当前 payload 关键结构：
  - `mode: "preview"`
  - `plan: { schema_version, mode, parameters, steps[] }`

### 执行状态（下一步建议补充）

- `engine.execution_started`
- `engine.execution_finished`
- `engine.execution_error`

当前未实现以上执行状态消息，待执行器接入后补齐。

## 里程碑

### M1（已完成）

- 动作原子提取与严格筛选（后端）
- 动作计划可视化与发送（前端）

### M2（进行中）

- 前端本地执行器：按计划驱动 Live2D 参数

### M3（待开始）

- 语义到计划自动生成
- expression/motion fallback 融合

## 验收标准（M2）

- 在桌宠窗口可看到 `engine.motion_plan` 对应的实际动作变化。
- 每次执行可输出开始/结束状态，失败可定位到具体通道或参数。
- 不依赖后端动作生成，也能完成本地组合动作播放。
