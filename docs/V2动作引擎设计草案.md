# V2 动作引擎设计草案

快照日期：2026-04-22

## 当前边界

- 动作引擎执行仍属于前端 runtime（Live2D WebSDK）。
- adapter 负责生成参数计划并通过协议下发，不做逐帧参数写入。
- 协议继续复用 `engine.motion_plan`，但 `plan.schema_version` 固定为 `engine.parameter_plan.v1`。

## 默认执行链（已切换）

1. 后端接收对话文本，LLM 生成 12 关键轴 + `emotion_label` + `duration_ms`
2. 后端基于 `parameter_action_library` 生成 `supplementary_params`
3. adapter 广播 `engine.motion_plan`（参数计划）
4. 前端 `usePreviewMotionPlayer` 严格校验计划后调用 Direct Parameter 接口
5. `LAppModel.update()` 每帧先跑 motion/physics/idle 基线，再叠加参数计划层，最后 `model.update()`

## 计划结构（强约束）

`plan` 固定结构：

- `schema_version`: `engine.parameter_plan.v1`
- `mode`: `expressive | idle`
- `emotion_label`: string
- `timing`: `{ duration_ms, blend_in_ms, hold_ms, blend_out_ms }`
- `key_axes`: 固定 12 轴，且每轴必须有 `{ value: 0..100 }`
- `supplementary_params`: `[{ parameter_id, target_value(-1..1), weight(0..1), source_atom_id, channel }]`

严格失败策略：

- 缺轴、越界、字段缺失、类型错误，整 plan 直接失败。
- 前端执行时任一参数不可写，整轮立即终止。
- 不走旧 `steps -> startMotion()` 兜底路径。

## 后端生成规则

- 活跃通道判定：`abs(axis - 50) >= 8`
- polarity：`axis > 50 => positive`, `< 50 => negative`
- 候选过滤：`primary_channel + polarity`
- 排序：`score desc, energy_score desc`
- 通道选取：Top2
- 去重：按 `parameter_id` 去重，保留高分项
- 排除：与 12 关键轴映射参数冲突项剔除
- `target_value`: 归一化语义偏移（`-1..1`）
- `weight`: 随轴强度线性增长，限制在 `0.15..0.9`
- 死区：12 轴全部在 `[42,58]` 时输出 `mode=idle`

## 前端混合规则

- 关键轴写入：`value(0..100)` 映射到参数 `min..max`，按 `timing` 做 blend-in/hold/blend-out
- 补充参数写入：增量叠加
  - `delta = target_value * 0.5 * range * weight * easing`
- Idle/motion/physics 作为基线保留，参数层仅做覆盖融合
