## 审阅发现

### 1. 配置热更新当前已经失效

- 状态：已修复
- 严重性：高
- 位置：
  - [plugin_runtime.py#L43](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\plugin_runtime.py#L43)
  - [runtime_state.py#L87](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\runtime_state.py#L87)
  - [runtime_state.py#L241](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\runtime_state.py#L241)
- 问题：
  - `get_plugin_config()` 返回的是普通 `dict`。
  - `RuntimeState.refresh()` 之后再调用 `_load_plugin_config_from_source()` 时已经拿不到 `config_path`。
  - 结果是 `client_uid`、`stt_provider_id`、VAD 参数、图片冷却时间等修改后不会被真正重新加载。
- 影响：
  - 当前代码里虽然保留了“刷新运行时配置”的入口，但实际只会重复使用启动时那份配置快照。

### 2. 纯图片回复不会正常结束 turn

- 状态：已修复
- 严重性：高
- 位置：
  - [turn_coordinator.py#L174](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\turn_coordinator.py#L174)
  - [turn_coordinator.py#L187](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\turn_coordinator.py#L187)
- 问题：
  - 当 AstrBot 回复只有图片、没有文本、也没有音频时，只会发送一次 `audio` 负载。
  - 当前分支不会发送 `backend-synth-complete`、`force-new-message`、`conversation-chain-end`。
  - 会话状态也不会回到空闲态。
- 影响：
  - 前端 UI 可能停留在未完成状态。
  - 下一轮交互可能受阻或表现异常。

### 3. `WebSocket` 入站 JSON 只要结构不对就可能直接断开

- 状态：已修复
- 严重性：中
- 位置：
  - [transport_ws.py#L127](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\transport_ws.py#L127)
  - [turn_coordinator.py#L69](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\turn_coordinator.py#L69)
- 问题：
  - 目前只校验了“是不是合法 JSON”，没有校验“是不是对象”。
  - 如果前端发来的是数组、字符串或数字，`handle_msg()` 内部的 `.get()` 会抛异常。
  - `_handle_client()` 没有为单条消息加保护，会直接掉连接。
- 影响：
  - 一条坏消息就能把当前前端连接打断。
  - 不利于前后端联调。

### 4. 传输层启动失败时，静态资源服务可能残留

- 状态：已修复
- 严重性：中
- 位置：
  - [transport_ws.py#L42](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\transport_ws.py#L42)
  - [transport_ws.py#L51](C:\Users\Administrator\Documents\GitHub\AG99live\adapter\adapter\transport_ws.py#L51)
- 问题：
  - 当前启动顺序是先启 `HTTP` 静态资源服务，再启 `WebSocket`。
  - 如果后续步骤抛异常，没有统一清理分支回收已启动的静态资源服务。
- 影响：
  - 二次启动时可能遇到端口占用。
  - 会增加插件异常恢复成本。
