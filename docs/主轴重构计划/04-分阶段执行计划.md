# 04 分阶段执行计划

## 总原则

这是一次破坏性重构。

目标不是长期兼容固定 12 轴，而是尽快把核心抽象迁移到动态主轴。

原则：

- 每个阶段独立提交。
- 每个阶段都有可验证结果。
- 不用兼容逻辑掩盖错误。
- 旧实现由 Git 保留，不在新架构里长期背负。

## 当前状态

- `Phase 0`：已完成
- `Phase 1`：已完成
- `Phase 2`：已完成
- `Phase 3`：已完成最小闭环
  - 已具备后端 canonical profile
  - 已具备前端读取与保存入口
- `Phase 4`：主体已完成
  - 已具备 Action Lab Profile Editor UI
  - 已能展示固定模板轴和扫描出的额外参数轴
  - 已支持编辑 bindings / couplings / semantics
  - 尚未完成扫描参数批量勾选 / 新建主轴交互
- `Phase 5+`：尚未开始核心迁移

## Phase 0：冻结当前状态

目标：建立回退点。

任务：

- 跑前端 `npm run typecheck`、`npm run build`。
- 跑后端测试。
- 记录当前动作链路状态。
- 提交当前文档和基线。

验收：

- 工作树干净。
- 有明确 commit 可回退。

## Phase 1：新增 SemanticAxisProfile 类型和默认 profile

目标：让代码拥有动态主轴的数据结构。

任务：

- 新增 profile 类型定义。
- 新增默认 profile，使用当前固定 12 轴作为初始模板。
- 标记控制权：
  - primary: head/gaze/eye expression/mouth_smile
  - derived: body_yaw/body_roll/brow_bias
  - runtime: mouth_open
- 不接入 compiler，只先建立数据结构。

验收：

- TypeScript 类型通过。
- 默认 profile 可被加载和展示。

## Phase 2：后端 Profile 存储、扫描回填与版本字段

目标：让后端成为 profile 的事实来源，并在模型目录中落盘。

任务：

- 在 `live2ds/<model_name>/ag99/semantic_axis_profile.json` 持久化 profile。
- 基于现有 model sync / scan 数据生成初版 `semantic_axis_profile`。
- 固定模板外的扫描参数先以 `debug` 轴进入 profile，供后续人工提升为 primary / derived / runtime。
- 计算 Live2D 文件夹 hash。
- 增加 `profile_id`、`revision`、`source_hash`、`user_modified` 等基础字段。
- hash 未变化时复用 profile。
- hash 变化时标记 profile stale，不静默覆盖用户修改。

验收：

- 当前 Mk6 模型可以生成或回填 profile。
- profile 以 JSON 持久化在模型目录。
- 扫描失败时有显式 error/warning。

## Phase 3：前后端 Profile 读写协议

目标：建立前端读取、编辑、保存后端 canonical profile 的最小协议。

任务：

- 新增 profile 同步协议或同步 payload。
- 前端读取当前模型 canonical profile。
- 前端保存编辑结果时显式提交给后端。
- 前后端使用同一组 `profile_id + revision` 做版本识别。
- profile 不一致时显式报 warning/error，不静默覆盖。

验收：

- 前端能读取当前模型 profile。
- 前端能把修改保存回后端模型目录。
- 后端和前端日志中能看到同一个 `profile_id` 和 `revision`。

## Phase 4：前端 Profile Editor / Action Lab 接入

目标：允许用户选择主轴和联动关系。

任务：

- Action Lab 新增 Profile Editor 区域。
- 动态展示 profile 中所有 scanned axes。
- 用户可修改：
  - control_role
  - label/description
  - positive/negative semantics
  - neutral/range
  - parameter binding
  - coupling 开关和强度
- 保存 profile 到后端。

验收：

- 不再依赖固定 12 轴生成 UI。
- 用户可以把某个参数设为 primary/derived/runtime/debug。
- 用户可以配置 `head_yaw -> body_yaw` 这类联动。

## Phase 5：ModelEngine v2 Compiler

目标：前端先具备消费动态 intent/profile 的能力。

任务：

- 新增 `motion_intent.v2` normalize。
- 新增动态 `axes: Record<string, { value: number }>`。
- 使用 profile 校验 axis id 和 value range。
- 派生 derived axes。
- 保留 diagnostics：
  - primaryAxes
  - derivedAxes
  - runtimeAxes
  - missingAxes
  - forbiddenAxes

验收：

- 动态 primary axes 可以编译。
- body/brow 等派生轴不需要 LLM 输出。
- 不再要求固定 12 轴完整存在。

## Phase 6：Prompt 生成改为 profile-driven

目标：后端 prompt 不再硬编码固定 12 轴。

任务：

- 后端接收或读取当前 profile。
- prompt 从 profile 中生成 primary/hint 轴说明。
- runtime/derived/ambient 轴不进入主输出列表。
- 输出 schema 切到 `engine.motion_intent.v2`。

验收：

- prompt 中不再出现固定 12 轴说明。
- LLM 输出动态 axes。
- forbidden axis 出现时解析层 warning/reject。

## Phase 7：ParameterPlan v2 Runtime

目标：Runtime 支持动态参数写入。

任务：

- 新增 `engine.parameter_plan.v2`。
- `parameters[]` 直接声明 `parameter_id/target_value/weight`。
- Runtime 只根据 parameter_id 写入，不再理解固定轴名。
- 写参失败必须报错或 warning，不静默吞。

验收：

- Live2D 可以执行 v2 plan。
- 不依赖 `DIRECT_PARAMETER_AXIS_NAMES`。
- Action Lab 可以回放 v2 plan。

## Phase 8：删除旧固定 12 轴核心路径

目标：清掉架构债务。

任务：

- 删除 `DIRECT_PARAMETER_AXIS_NAMES` 作为核心依赖。
- 删除 v1-only compiler 依赖。
- 删除固定 12 轴 UI。
- 删除固定 12 轴 prompt 说明。
- 文档标记 v1 为历史。

验收：

- 生产链路只走 profile-driven v2。
- 搜索固定 12 轴常量，不再是核心路径。
- 测试和构建通过。

## 推荐提交序列

```text
chore: document dynamic semantic axis rewrite plan
refactor: add semantic axis profile model
feat: generate semantic axis profile from model scan
feat: add semantic axis profile editor
feat: sync semantic axis profile across frontend and backend
feat: compile dynamic motion intent v2
refactor: generate motion prompt from semantic profile
feat: play dynamic parameter plan v2
refactor: remove fixed twelve-axis motion path
```

## 验收总标准

重构完成时必须满足：

- 不再把固定 12 轴作为架构核心。
- 主控参数由 profile 决定。
- prompt 根据 profile 生成。
- 前端可编辑主轴和联动关系。
- ModelEngine 可编译动态 axes。
- Runtime 可执行动态 parameter plan。
- 所有派生/runtime/ambient 控制权可诊断。
