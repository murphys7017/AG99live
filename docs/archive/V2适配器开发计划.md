# V2 适配器开发计划

快照日期：2026-04-21

## 定位

V2 适配器是 AstrBot 插件侧的基础设施层：

- 消息协议桥接
- 语音/文本/媒体链路
- Live2D 资源扫描与基础产物下发

不承担前端实时动作引擎职责。

## 阶段状态

### Phase 1：消息适配底座迁移（已完成）

- 已完成 V1 可用链路迁移到新插件根目录 `adapter/`
- 已完成 V2 协议主链路
- 已完成 WS + HTTP 双链路
- 已完成历史、背景、心跳等系统消息迁移

### Phase 2：扫描与基础动作库（已完成首版）

- 已完成模型资源扫描
- 已完成参数/expression/motion 基础解析
- 已完成 `base_action_library` 产出
- 已完成规则初筛 + LLM 严格筛选 + fallback 回退

### Phase 3：动作执行协作（进行中）

- 已接入 `engine.motion_plan` 协议占位
- 未完成真实动作执行协作协议（当前仅联调入口）

## 当前能力清单

### 通信与消息

- `input/output/control/system` 主链路可用
- `engine` 仅骨架与预览入口
- 所有消息采用统一 envelope

### 语音与媒体

- 音频流输入、麦克风链路、STT 转写
- 音频缓存与 URL 下发
- 图片输入落盘与冷却策略

### 模型与扫描

- 多模型发现与索引
- `model_sync` 下发模型完整信息
- `base_action_library`、`motion_resource_pool`、`constraints` 已可下发

## 配置项（新增重点）

- `enable_action_llm_filter`
- `action_llm_filter_timeout_seconds`
- `action_llm_filter_min_selected_channels`
- `action_llm_filter_max_atoms_per_channel`

## 部署与同步

- 适配器以 AstrBot 插件结构存在（含 `metadata.yaml`）
- 仓库内已提供部署脚本：`deploy_adapter.ps1`
- 当前同步策略：后端代码 + Live2D 后端资源

## 下一步计划

1. 为 `engine` 执行回执补正式消息（开始/完成/失败）。
2. 补协议回归测试，覆盖非法消息与断连恢复。
3. 收敛低价值兼容路径，减小维护负担。
4. 配合前端执行器落地，补充最小动作协作字段。
